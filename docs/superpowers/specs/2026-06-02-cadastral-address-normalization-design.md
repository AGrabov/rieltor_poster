# Дизайн: нормалізація адреси (RU→UA) + строгий пошук кадастру

**Дата:** 2026-06-02
**Статус:** затверджено до реалізації

## Проблема

CRM зберігає частини адреси непослідовно — інколи російською:

- **Тип вулиці** російською: `ул.`, `пер.` (замість `вул.`, `пров.`).
- **Назва вулиці** російською: `Пушкина` замість `Пушкінська`.
- **Місто** російською: `Киев` замість `Київ`.

Жодної валідації під час заповнення БД немає, тому «брудний» адрес потрапляє і
в `offers.db`, і в запит до реєстру кадастру.

### Що це ламає (перевірено вживу через `api.zem.center` 2026-06-02)

- **Тип вулиці в запиті вбиває пошук:** `Київ пер. Лугова 5` → **0** результатів,
  `Київ Лугова 5` → **6**.
- **Російське місто вбиває пошук:** `Киев Львовская 19` → **0**,
  `Київ Львівська 19` → **3**.
- **Рушій терпить и↔і:** `Київ Пушкинська 1` (рос. «и») знайшов `Пушкінська`.
  Тобто різниця алфавіту в назві вулиці часто не заважає — *якщо місто українське
  і тип вулиці зрізано*.
- **Латиниця не працює** (`Lvivska 19` → 0). Морфологічно російські назви
  (`Львовская`→`Львівська`) рушій не зіставить.

Додатково: нинішній `_pick_by_house` у разі відсутності збігу повертає
**першого кандидата** (`candidates[0][0]`), тобто може заповнити **чужий**
кадастровий номер. Чернетку на rieltor.ua після збереження **не можна
відредагувати** — ані кадастр, ані адресу. Тому хибне заповнення гірше за порожнє.

## Рішення (загальний підхід)

«Брати тільки назву вулиці й номер будинку, нормалізувати місто, а в реєстрі
серед запропонованих варіантів шукати точний збіг». Якщо впевненого збігу немає —
**не заповнювати**.

Реалізується трьома частинами + спільний модуль нормалізації.

---

## 1. Спільний модуль `crm_data_parser/address_normalize.py`

Єдине джерело правди для нормалізації адреси. Замінює два нинішні дублюючі
`_strip_street_prefix` (у `html_parser.py` і `cadastral_lookup.py`).

### Публічний API

```python
RU_UA_CITY: dict[str, str]          # folded рос. назва → укр. назва

def normalize_city(city: str) -> str:
    """Зрізати 'г./м./город/місто', знайти у RU_UA_CITY, інакше повернути як є."""

def strip_street_type(street: str) -> str:
    """Зрізати тип вулиці (UA + RU форми) з початку рядка."""

def fold_cyrillic(s: str) -> str:
    """Звести рядок до форми для ПОРІВНЯННЯ (не для відображення):
    ы→и, э→е, ё→е, ъ→'', уніфікувати и/і, прибрати апострофи, lower, trim."""
```

### Деталі

- **`RU_UA_CITY`** — засіяти містами, де реально працюють (мінімум):
  Київ, Харків, Одеса, Дніпро, Львів, Запоріжжя, Вінниця, Миколаїв,
  Кривий Ріг, Чернігів, Черкаси, Житомир, Суми, Хмельницький, Чернівці,
  Рівне, Івано-Франківськ, Тернопіль, Луцьк, Ужгород, Кропивницький,
  Полтава, Бровари, Бориспіль, Ірпінь, Буча, Біла Церква.
  Ключі зберігати у `fold_cyrillic`-формі, lookup — теж через `fold_cyrillic`,
  щоб `Киев`/`КИЕВ`/`киев` усі знаходились. Легко доповнюється.
- **`strip_street_type`** — типи UA+RU, довші альтернативи перед коротшими
  (щоб `пров.` зрізалось раніше за `пр`):
  `вулиця/вул./ул.`, `провулок/пров./пер./переулок`,
  `проспект/просп./пр-т/пр-кт/пр./просп`, `бульвар/бул./бульв./б-р`,
  `площа/пл./площадь`, `шосе/шос./ш./шоссе`, `набережна/наб./набережная`,
  `тупик/туп.`, `узвіз`, `проїзд/проезд`, `мікрорайон/мкр./мкрн`,
  `квартал/кв-л`. Регістронезалежно, із крапкою й без.
- **`fold_cyrillic`** — тільки для порівняння (вибір кандидата), ніколи не пишеться
  ні в БД, ні в запит відображення.

## 2. Парсер `crm_data_parser/html_parser.py`

При розборі адреси:

- `Місто` → `normalize_city(value)`.
- `Вулиця` → `strip_street_type(value)` (через спільний модуль; прибрати локальний
  `_STREET_PREFIXES`/`_strip_street_prefix`).

Так у `offers.db` лягає чистий український адрес. Бонус: автокомпліт адреси на
rieltor.ua (місто/вулиця — MUI autocomplete) теж перестане спотикатись на
російських написаннях.

Наявні рядки БД лишаються «брудними» до перепарсингу — їх підстраховує крок 3
(нормалізація на льоту в запиті). Міграцію наявних рядків НЕ робимо (YAGNI).

## 3. Пошук кадастру `crm_data_parser/cadastral_lookup.py`

### Побудова запиту

Замість локального `_strip_street_prefix` використати спільний модуль:

```
q = normalize_city(місто) + " " + strip_street_type(вулиця) + " " + будинок
```

(і варіант без будинку, як зараз). Підстраховує і старі рядки БД, де адрес ще
російський.

### Строгий вибір кандидата (ключова зміна)

`_pick_by_house` → перейменувати за змістом (напр. `_pick_verified`). Заповнюємо
кадастр **лише** якщо серед кандидатів є такий, що задовольняє **ОБИДВІ** умови:

1. **Номер будинку** збігається як окремий токен (точний збіг, як нинішній
   `exact_re`: `19`, не `19-а`).
2. **Назва вулиці** фуззі-збігається: `difflib.SequenceMatcher.ratio()` між
   `fold_cyrillic(назва-вулиці-із-запиту)` та назвою вулиці кандидата
   (з `fold_cyrillic`), поріг **~0.75–0.8** (фінал добити тестами).
   Назву вулиці кандидата дістаємо з `address`-рядка зрізанням типу
   (`strip_street_type`) і відкиданням хвоста з номером/районом — або
   partial-ratio по токенах адреси (деталь у плані реалізації).

Якщо жодного кандидата з обома умовами немає → повертаємо `None`, поле
**не заповнюється**.

### Прибрати небезпечні фолбеки

- `return candidates[0][0]` (перший валідний) — **видалити**.
- «порожній house → перший кандидат» — **видалити** (порожній дім → не можемо
  верифікувати → `None`).

Решта ланцюжка (zem.center → kadastrova-karta fallback, крок-0 regex по опису)
лишається; обидва джерела проходять через той самий строгий `_pick_verified`.

Публічні сигнатури `lookup_cadastral_number`, `enrich_offer_data_with_cadastral`,
`fill_missing_cadastral_numbers` **не змінюються**.

## 4. Тести

**Новий `tests/test_address_normalize.py`:**
- `normalize_city`: `Киев`→`Київ`, `г. Киев`→`Київ`, `Харьков`→`Харків`,
  невідоме місто → без змін, вже-українське → без змін.
- `strip_street_type`: `ул. Лугова`→`Лугова`, `пер. Садовий`→`Садовий`,
  `пров. Садовий`→`Садовий`, `вулиця Львівська`→`Львівська`, без типу → без змін.
- `fold_cyrillic`: и↔і однакові, ы→и, э→е, апострофи прибрано, регістр.

**Доповнення `tests/test_cadastral_lookup.py`:**
- Строгий матч: дім + вулиця збігаються → cadnum; дім збігається, вулиця ні → `None`;
  вулиця збігається, дім ні → `None`.
- и↔і толерантність: запит `Пушкинська`, кандидат `Пушкінська`, дім збігається →
  заповнюється.
- Небезпечний фолбек прибрано: список кандидатів без збігу дому → `None`
  (раніше повертав першого).
- Порожній house → `None`.
- Фікстури JSON (zem.center) і HTML (kadastrova-karta) — без мережі, через
  monkeypatch.

## Що НЕ робимо (YAGNI)

- Повноцінну RU→UA транслітерацію назв вулиць (морфологія `Львовская`→`Львівська`).
  Такі рідкісні випадки просто пропускаються (безпечно — краще порожнє, ніж хибне).
- Латиничні запити (рушій їх не підтримує).
- Міграцію/перепарсинг наявних рядків БД — підстраховує нормалізація на льоту.
- Нові залежності — досить stdlib `difflib`.
- Зміну публічних сигнатур функцій кадастру.
